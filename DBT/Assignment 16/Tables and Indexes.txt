1) Write a command that will enable a user to pull orders grouped by date out of the
Orders table quickly.
=> Select Odate, Count(*) from orders
	Group by Odate;		// Gives values in 0.007 sec

Create index i_Date on Orders(Odate);
// After Creating index gives values in 0.033 sec

// It is slow after creating index is because if rows in table is very less in that condition 
   Full table scan is efficient than index but it will be faster when there are large number of rows.

be unique (assume all current values are unique)?

   inside column is unique or not if not then it will give an error.
	
	// Command to store Unique values in Onum column
